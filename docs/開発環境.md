# 開発環境（Windows / Linux）

クイックスタート・`uv sync` の組み合わせ例・CLI 一覧は [README.md](../README.md) を参照してください（本書はコンテナ・GPU・各 extra の詳細に寄せています）。

## 方針

| 環境 | 推奨 |
|------|------|
| **Windows** | **Dev Container**（本リポジトリの `.devcontainer`）。Linux コンテナ上で CUDA 12.8 用 PyTorch と依存関係を揃えやすいです。 |
| **Linux** | **そのまま**ホストに NVIDIA ドライバと `uv` を入れ、リポジトリ直下で `uv sync` すれば同じ `pyproject.toml` / `uv.lock` が使えます。 |

PyTorch は **`pyproject.toml` の `[[tool.uv.index]]`（PyTorch 公式 `cu128`）** 経由で入り、**CUDA 12.x 向けビルド**になります（ご利用の **CUDA 12.8** ドライバ環境向け）。

---

## Windows（Dev Container）

### 前提

- [Docker Desktop](https://www.docker.com/products/docker-desktop/)（WSL2 バックエンド推奨）
- NVIDIA ドライバ + Docker からの GPU 利用（WSL2 で GPU が使える設定）
- VS Code または Cursor の **Dev Containers** 拡張

### 手順

1. リポジトリをクローンし、エディタでフォルダを開く。
2. **「Dev Containers: Reopen in Container」** を実行する。
3. コンテナ作成後、`postCreateCommand` で **`uv sync --extra sidon --extra dev`** が走り、`.venv` が作られる（[`config/default.yaml`](../config/default.yaml) の Sidon 既定に合わせた構成）。
4. **NeMo Forced Aligner（NFA）** 用に、イメージビルド時に **`/opt/nfa-venv`**（`torch` / `torchaudio` cu128 + `nemo_toolkit[asr]`）と **NeMo ツリー**（`/opt/NeMo/tools/nemo_forced_aligner` の `align.py`）が入る。`.venv` には入れない（**`sgmse` extra の protobuf 要件と NeMo が衝突する**ため）。環境変数 **`NFA_PYTHON`**, **`NFA_ALIGN_DIR`** がコンテナに設定される。インタープリタ（編集・pytest）は `/workspace/.venv/bin/python`（設定済み）。

GPU が無いマシンでもビルドは通る想定です（`hostRequirements.gpu` は optional）。学習・Sidon / Dasheng 等の推論で GPU が必要な場合は Docker の GPU 設定を有効にしてください。

### NeMo Forced Aligner（NFA、`nfa_gate`、任意）

**Dev Container** では **MFA（conda）は入れていません**。前処理の **`nfa_gate`** は [NeMo Forced Aligner](https://docs.nvidia.com/nemo-framework/user-guide/latest/nemotoolkit/tools/nemo_forced_aligner.html) の `align.py` を **`NFA_PYTHON` で実行**します（`NFA_ALIGN_DIR` をカレントディレクトリにして `align.py` を起動する形が確実です）。

手動例（コンテナ内・GPU 想定）:

```bash
cd "$NFA_ALIGN_DIR"
"$NFA_PYTHON" align.py \
  pretrained_name=nvidia/parakeet-tdt_ctc-0.6b-ja \
  manifest_filepath=/abs/path/to/manifest.json \
  output_dir=/abs/path/to/out
```

マニフェストは NeMo 形式の JSONL（各行に **`audio_filepath`（絶対パス）** と **`text`**）。参照モデル例 **`nvidia/parakeet-tdt_ctc-0.6b-ja`** は **16 kHz** 入力の hybrid（NFA は **CTC デコード**に切替）。パイプライン本流への組み込みは [docs/仕様.md](./仕様.md) §5.3 を参照。

### Montreal Forced Aligner（`mfa_gate`、ホストのみ）

Dev Container には **含めていません**。`mfa_gate` を MFA で使う場合は、[MFA 公式](https://montreal-forced-aligner.readthedocs.io/en/latest/installation.html)のとおり **conda-forge の `montreal-forced-aligner`** 等をホスト環境に入れ、`mfa` が `PATH` で解決するようにしてください。`pyproject.toml` の optional **`mfa`** extra は宣言用であり、**kalpy は別途 conda が必要**です。

### HiFi-GAN 帯域補完（`bandwidth_extension`、任意）

二次パイプラインの音声ステップで `type: bandwidth_extension` を使う場合、**PyTorch** が必要です。

```bash
uv sync --extra hifigan
# Sidon 既定（default.yaml）に HiFi-GAN を足す例
uv sync --extra sidon --extra hifigan --extra dev
```

[jik876/hifi-gan](https://github.com/jik876/hifi-gan) の学習済み **Generator** 重みと同じディレクトリ（または `--config_json` 相当）の **`config.json`** を取得し、`generator_checkpoint` にパスを指定してください（詳細は `docs/追加仕様.md` §12）。

### Common Voice データ（ホストをマウント）

既定では、ワークスペースの **1 つ上の階層の `dataset/CommonVoice`** をコンテナ内の **`/workspace/CommonVoice`** にバインドします。

| ホスト例（Windows） | コンテナ内 |
|----------------------|------------|
| `…\Fio-TTS\FilteringCV` を開いているとき `…\Fio-TTS\dataset\CommonVoice` | `/workspace/CommonVoice` |

`config/default.yaml` の `input.corpus_root: CommonVoice` はリポジトリルート相対なので、このマウントでそのまま参照できます。

データを別の場所に置いている場合は `.devcontainer/devcontainer.json` の `mounts` の **`source=`** を、ホストの絶対パス（Docker が読める形式）に書き換えてください。フォルダが存在しないとコンテナ起動時にマウントで失敗することがあります。

## Linux（ホストでそのまま）

### 前提

- NVIDIA ドライバ（CUDA 12.8 利用に十分なバージョン）
- [uv](https://docs.astral.sh/uv/getting-started/installation/) をインストール

### 手順

```bash
cd /path/to/this/repo   # 本リポジトリのルート
uv sync --extra sidon --extra dev
source .venv/bin/activate   # 任意
cv-preprocess --help
```

**MFA 足切り**をホストで使う場合は、上の **「Montreal Forced Aligner（`mfa_gate`、ホストのみ）」** を参照してください。

`pyproject.toml` の `[tool.uv.sources]` により **torch / torchaudio は PyTorch の cu128 用ホイール**から入ります。CPU のみの環境では別途インデックス変更や CPU 用 torch の入れ方が必要です。

---

## SGMSE（`speechbrain/sgmse-voicebank`、任意）

[`config/default.yaml`](../config/default.yaml) の既定 enhance は **Sidon のみ**で、SGMSE は含みません。`denoise.method: sgmse` をパイプラインに追加するときに次を参照してください。

`config` で `denoise.method: sgmse` を使う場合:

1. `uv sync --extra sgmse` — torch / torchaudio / speechbrain / **torch-ema** / **ninja** に加え、
   `pyproject.toml` の `[tool.uv.sources]` で **GitHub 版 `sgmse`**（`sp-uhh/sgmse`）が入る（手動の
   `uv pip install "sgmse @ git+..." --no-deps` は通常不要）。
2. **JIT ビルド用ツールチェーン**（次が無いとビルドは失敗します）:
   - **g++**（例: `build-essential`）
   - **Python.h**（例: `python3-dev`。無いと `fatal error: Python.h: No such file or directory` になる）
   - **nvcc**（CUDA Toolkit。**`nvidia/cuda:*-runtime-*` の Docker イメージには nvcc が含まれません**。本リポジトリの Dev Container は **`devel`** ベースにして nvcc を同梱しています）
   - **ninja**（手順 1 の `sgmse` extra に含まれる）

3. `pyproject.toml` の `[[tool.uv.dependency-metadata]]` で、Git 版 `sgmse` の公式メタデータに付く学習用の巨大な依存をロック解決から外し、推論用に extra へ載せたパッケージだけを使う構成にしています。

---

## NARA-WPE + DeepFilterNet（`wpe_dfn`、任意）

`config` で `denoise.method: wpe_deepfilternet` を使う場合:

1. `uv sync --extra wpe_dfn` — `nara-wpe` / `deepfilternet` / `torch` / `torchaudio`。
2. **deepfilterlib**（Rust）のビルドに **cargo / rustc** が必要な環境がある（Dev Container の Dockerfile に `rustc` / `cargo` を入れています）。
3. 学習済み重みは初回に GitHub の `models/*.zip` から取得（**MIT / Apache-2.0 デュアル**、商用可。利用条件は各ライセンスに従うこと）。`deepfilternet_model` で `DeepFilterNet` / `DeepFilterNet2` / `DeepFilterNet3` またはローカル展開ディレクトリを指定。

## Sidon（`sidon_restore` / `sidon_after_enhance_split`）

[`config/default.yaml`](../config/default.yaml) では **enhance** の **`type: sidon_restore`** と **`quality_gate.sidon_after_enhance_split`** が既定で有効です。Dev Container / 推奨ホスト同期は次のとおりです。

```bash
uv sync --extra sidon --extra dev
```

初回推論で **Hugging Face**（`sarulab-speech/sidon-v0.1` 等）から重み・SSL 特徴抽出器を取得します。Dasheng や SGMSE を同じ設定に含める場合は [README.md](../README.md) のセットアップ節のとおり **`--extra dasheng`** / **`--extra sgmse`** を追加してください。

---

## ロックファイル

依存変更後はリポジトリ管理者が **`uv lock`** を実行し、`uv.lock` をコミットしてください。Dev Container の `postCreateCommand` は通常 `uv sync` のみです（ロックはリポジトリに含まれる想定）。
