# ASR ベース音素崩れフィルタ（Parakeet Gate）拡張仕様

本書は **FilteringCV / cv-preprocess**（CLI: `cv-preprocess`）に、Parakeet 系 ASR を用いた **逆認識による音素・テキスト整合ゲート**（以下 **`asr_gate`**）を追加する前提の設計ドラフトです。

**2026-05 調整**: 本リポジトリの **現行実装**（`cv_preprocess/config/pipeline.py` 等）と突き合わせ、設定キーの置き方・既存機能との関係・モデル名の扱いを修正しました。`asr_gate` 本体は **未実装**です。

---

## 0. 本リポジトリ現状との対応表

| 本書の要素 | 現状（このリポジトリ） | メモ |
|------------|------------------------|------|
| **`asr_gate`** | **未実装** | 追加時は `PipelineConfig` の **トップレベル**に新ブロックを置く（`mfa_gate` / `nfa_gate` と同列）。 |
| **`quality_gate:`（YAML）** | **`QualityGateConfig` のみ**（長さ・無音率・SNR・モーラ・ティア等） | MFA/NFA/ASR 用の子ブロックは **入れない**。本書初版の「`quality_gate.asr_gate`」は **スキーマと不一致**のため廃止。 |
| **`mfa_gate` / `nfa_gate`** | 実装済み（`cv_preprocess/config/align_gates.py`） | **同時 `enabled: true` 不可**（`PipelineConfig.mfa_nfa_exclusive`）。 |
| **Parakeet** | **`nfa_gate.pretrained_name` の NeMo チェックポイント**として使用 | 既定例は **`nvidia/parakeet-tdt_ctc-0.6b-ja`**（16 kHz、`model_sample_rate_hz`）。**強制アライメント（NFA）** 用。 |
| **`pred_text` と参照 G2P の音素距離** | **`nfa_gate.align_using_pred_text`** + **`compare_pred_text_to_norm`** | 認識に近い経路だが、あくまで **NFA パイプライン内**のオプション。純粋な「ASR のみバッチ推論 → 閾値」ではない。 |
| **`text.phoneme_alignment_check`** | 実装済み（`gates_quality.py`） | 外部 JSONL マニフェストと G2P の **`max_token_error_rate`** 照合。 |
| **`phoneme-manifest`** | 実装済み（`cv-preprocess phoneme-manifest`） | `phoneme_alignment_check` 用マニフェスト生成。 |
| **G2P** | **`pyopenjtalk-plus`**（`pyproject.toml` の依存） | `g2p_phonemes`（`cv_preprocess/text/phonemize.py`）。 |
| **参照文の正規化** | パイプラインでは主に **`normalize_for_tts`**（`cv_preprocess/text/normalize.py`） | NFKC・空白整理・軽い句読点処理など。本書 §5.1 の詳細正規化は **`asr_gate` 側で拡張**するか、既存関数を共有するかを実装時に決める。 |
| **メタデータ TSV** | **`input.clip_tsv`**（例: `validated.csv`） | 列 `sentence` は Common Voice 慣習。拡張子は設定依存。 |
| **話者単位集計・話者ゲート** | **`asr_gate` 相当は未実装** | `report.json` に NFA 設定サマリはあるが、**`client_id` 別の CER/PER 集計による除外**は本書 §8 の新規スコープ。 |
| **デバッグ用 CLI** | **`cv-preprocess suggest-nfa-g2p-map`** / **`suggest-mfa-g2p-map`** が既存 | 本書の `suggest-asr-g2p-map` は **任意の将来案**として維持。ASR 出力が素のテキストなら、トークンマップより **`phoneme-manifest` と同型の音素列比較**で足りる想定。 |

## 1. 目的

Common Voice 由来の音声から、TTS 学習に不向きな発話を除外する。特に以下を検出対象とする。

* 読み飛ばし
* 文字起こしとの不一致
* 子音脱落
* 無声化・連結崩れ
* 早口・詰まり・言い直し
* 話者癖による音素不安定

無音除去や SNR 評価では拾い切れない **テキストと実発話のズレ** を弾くことを主目的とする。

---

## 2. 位置づけ（実装時）

* **`preprocess`** の品質判定レイヤに **`asr_gate` を追加**する想定。既存の **`mfa_gate` / `nfa_gate`** と同様、**設定で on/off** し、拒否理由を **`rejects.csv` / `report.json`** に積めること。
* 本機能は **強制アライメントではなく、ASR 逆認識（またはそれに準ずるデコード結果）による品質判定** を行う。NFA の `pred_text` 照合（§0 表）と **役割が重なる**ため、実装時は **併用可否・実行順序・二重拒否**を設計する。
* **`phoneme_alignment_check`** は「別プロセスで得た音素列と G2P を比較する機能」として残し、**`asr_gate` と独立**に扱う（併用可）。

---

## 3. 処理フロー（論理）

```text
input.clip_tsv（例: validated.csv）
  ↓
（既存）テキスト正規化・ロケール/日本語/長さ・任意で phoneme_alignment_check
  ↓
（既存）音声パイプライン（align / enhance・品質ゲート等）
  ↓
ASR 推論（Parakeet 系・実装時にバックエンド固定または選択）
  ↓
推論テキスト正規化
  ↓
参照テキスト正規化
  ↓
文字列比較 / 音素比較
  ↓
スコア算出
  ↓
閾値判定
  ↓
reject / accept
```

推奨の評価順（論理）は次のとおり。

1. 音声正規化（対象波形は **既存パイプラインのどの段の出力に載せるか**を実装時に固定。例: align 直後の WAV 相当など）
2. ASR 推論
3. テキスト正規化（参照・仮説の両方）
4. G2P 音素化（`text.phonemize: true` が前提になりうる）
5. 音素距離計算（必要なら重み付き）
6. 話者単位集計（§8）
7. 最終判定

---

## 4. 新規設定ブロック（案）

**`PipelineConfig` 直下**に `asr_gate` を追加する形を推奨する（`quality_gate` 配下ではない）。

```yaml
# 例: config/default.yaml と同列（mfa_gate / nfa_gate の近く）
asr_gate:
  enabled: false
  backend: parakeet
  # 実装時: NeMo / ONNX 等で利用可能なチェックポイント名またはローカルパス。
  # 本リポジトリの NFA 既定は parakeet-tdt_ctc-0.6b-ja。純粋 ASR 用は別チェックポイントを選ぶ前提。
  model_name: nvidia/parakeet-tdt-0.6b-v3
  sample_rate_hz: 16000

  min_asr_confidence: 0.70
  max_char_error_rate: 0.20
  max_phoneme_error_rate: 0.25

  compare_text: true
  compare_phonemes: true

  normalize_reference_text: true
  normalize_hypothesis_text: true
  # 実装時: 既存 normalize_for_tts を使う / 拡張する / 別正規化器を指す、のいずれかを明示
  text_normalizer: openjtalk_like

  persistent_worker: true
  batch_size: 8
  num_workers: 1

  missing_transcript: reject
  decode_failure: reject
```

Pydantic モデルは **`cv_preprocess/config/`** に `AsrGateConfig` 等として追加し、`PipelineConfig` に `Field(default_factory=...)` を載せる。 **`mfa_gate` / `nfa_gate` との排他**が必要なら `model_validator` で検証する。

---

## 5. 判定ロジック

### 5.1 参照テキスト

* 基準文は **`input.clip_tsv` の `sentence` 列**（Common Voice 慣習）。正規化は比較前に行う。
* 正規化の具体（全角半角、記号、数字読み、かな揺れ抑制など）は、既存の **`normalize_for_tts`** の範囲を超える場合は **`asr_gate` 専用オプション**として定義する。

### 5.2 ASR 出力

* 仮説文に対して **CER**、（任意で）**ASR confidence**、短すぎ/長すぎの異常長を計算する。

### 5.3 音素比較

* 参照文と ASR 出力を **同一 G2P パイプライン**（本リポジトリでは **OpenJTalk / `pyopenjtalk-plus` 経由の `g2p_phonemes`**）で音素化し、編集距離ベースの **PER（音素誤り率）** を計算する。
* 複合条件の例: `CER <= 0.20` かつ `PER <= 0.25` かつ `confidence >= 0.70`（閾値はデータで調整）。

### 5.4 重み付き音素（任意）

以下の音素カテゴリは崩れが TTS で目立ちやすいため、距離に重みを付けられるようにする（実装優先度は低めで可）。

* `s` / `sh` / `ts` / `ch` / `t` / `r` / `N` / 促音 等

---

## 6. 拒否理由コード（案）

クリップ単位で追跡できる **`asr_*` プレフィックス**を推奨する（既存の `nfa_*` / MFA 理由と衝突しないようにする）。

* `asr_decode_failed`
* `asr_low_confidence`
* `asr_text_mismatch`
* `asr_phoneme_mismatch`
* `asr_duration_outlier`
* `asr_missing_transcript`
* `asr_worker_error`

**`report.json`** には、既存の `nfa_gate` ブロックと同様、**設定サマリ・件数集計**を載せる方針に合わせる（`cv_preprocess/pipeline/preprocess/session.py` の `report` 組み立てを拡張）。

---

## 7. 出力項目（案）

### 7.1 クリップ単位

* `asr_hypothesis`
* `asr_confidence`
* `char_error_rate`
* `phoneme_error_rate`
* `asr_decision`
* `asr_reject_reason`

（メタデータ JSONL への載せ方は、既存の accepted クリップメタフィールドに合わせて追加。）

### 7.2 集計（話者単位）

* 平均 CER / PER / confidence
* 失敗率
* 危険音素比率（§5.4 を採用した場合）
* 平均話速 等

---

## 8. 話者単位ゲート（案）

`client_id` 単位で集計し、平均 PER が高い・特定音素誤りが多い・失敗率が高い・話速分散が大きい話者を **優先的に除外**する。Common Voice における **不安定話者の塊**対策。現行コードベースには **同等機能なし**（本書スコープの新規実装）。

---

## 9. 実行方式

* **推奨**: `persistent_worker: true` で **モデル初回ロードのみ**、クリップは **バッチ推論**。既存 **`nfa_gate.persistent_worker`**（`cv_preprocess/audio/nfa_align_worker.py`）と同じ思想。
* **`persistent_worker: false`**: バッチ（またはジョブ）ごとにプロセス起動。
* **worker 例外**: バッチ失敗扱いとするか、クリップ単位で `asr_worker_error` に落とすかを実装時に仕様化。
* **`decode_failure`**: 原則 **reject**。

---

## 10. CLI

* 本流は **`cv-preprocess preprocess -c <config.yaml>`** の **設定のみ**で制御（新サブコマンドは必須にしない）。
* デバッグ用の **`cv-preprocess suggest-asr-g2p-map`** は **任意**（既存の `suggest-nfa-g2p-map` と対になるかどうかは、ASR トークン列を比較する必要が出た時点で判断）。

---

## 11. `phoneme_alignment_check` との関係

| 機能 | 役割 |
|------|------|
| **`phoneme_alignment_check`** | 外部生成の音素列（マニフェスト）と G2P の比較（`manifest_path` / `missing_manifest_entry` / `max_token_error_rate`）。 |
| **`asr_gate`（本書）** | ASR 逆認識に基づく参照文との整合。 |

併用可能だが、**データソースと拒否理由**を混同しないこと。

---

## 12. 受入基準

* 明らかな読み飛ばしが除外率向上で確認できる
* サ行・タ行などの崩れが **意図的に**高率に拾える（閾値・重みで調整）
* 話者単位の不安定塊が除去できる（§8 を実装した場合）
* **`report.json`** に ASR 系の集計が出る
* 既存の **`mfa_gate` / `nfa_gate` / `phoneme_alignment_check` / `phoneme-manifest`** を壊さない

---

## 13. 実装優先順位（案）

1. `AsrGateConfig` と **`PipelineConfig` 統合**（既定 `enabled: false`）
2. ASR worker（常駐／バッチ）と推論インタフェース
3. 正規化・CER/PER・confidence 算出
4. `report.json` / rejects / メタデータ拡張
5. 話者集計・話者ゲート（§8）
6. 閾値調整用の検証コーパス整備
